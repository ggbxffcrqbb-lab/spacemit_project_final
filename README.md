# 基于端侧大模型与 RAG 架构的全离线多模态智能防腐专家交互终端

本仓库为本作品的竞赛最终提交版源码工程。

作品面向油气管网、海洋装备、化工储罐等工业防腐巡检场景，针对偏远无网、现场双手被占用、专业知识获取门槛高、排障决策滞后等问题，基于 `K1 Muse Pi Pro` 板端平台，构建了一套全离线、多模态、可交互的智能防腐专家终端。

系统在板端本地完成语音识别、视觉巡检、RAG 知识增强、大模型问答、语音播报和屏幕图文联动，形成“感知 - 推理 - 执行”的完整闭环，不依赖云端 API。

## 一、竞赛信息

- 赛题方向：进迭时空赛题 - 边缘 AI 应用
- 技术平台：`K1 Muse Pi Pro`
- 作品名称：基于端侧大模型与 RAG 架构的全离线多模态智能防腐专家交互终端
- 作品形态：板端离线语音 + 视觉巡检 + RAG 问答 + 语音播报 + 图文联动

## 二、作品目标

本作品重点解决以下真实问题：

1. 工业防腐巡检现场常处于无网或弱网环境，无法依赖云端能力。
2. 巡检人员现场操作时双手常被占用，传统交互方式不方便。
3. 防腐缺陷识别、风险判断和处置建议高度依赖专业经验。
4. 巡检过程需要可解释、可追溯、可展示的智能辅助能力。

因此，系统设计目标是：

1. 在板端全离线运行关键 AI 能力。
2. 打通语音、视觉、知识库与播报输出的多模态闭环。
3. 面向真实防腐场景提供自然交互与专业辅助决策能力。
4. 满足边缘 AI 赛题对推理性能、交互体验、资源管理和系统稳定性的要求。

## 三、核心功能

本作品最终实现了以下能力：

1. 离线语音唤醒与语音问答。
2. 双场景智能巡检切换。
3. 原生视频预览与检测框叠加显示。
4. 一级视觉检测加二级细分类判定。
5. 防腐领域本地 RAG 知识增强问答。
6. 语音播报与屏幕图文联动。
7. 中文快捷语音指令，如“总结一下”“保存截图”“退出程序”。
8. 面向竞赛展示的实时信息面板，包括最近问答、知识库命中和性能指标。

## 四、系统特点

### 1. 全离线

- 不依赖任何云端 API。
- 关键能力全部在 `Muse Pi Pro (K1)` 板端本地运行。

### 2. 多模态

- 语音输入
- 视觉输入
- 知识库增强
- 语音输出
- 屏幕图文输出

### 3. 面向真实场景

- 针对工业防腐巡检任务设计
- 强调现场交互便利性
- 强调知识辅助与结果解释能力

### 4. 竞赛展示友好

- 原生预览更流畅
- 双级检测结果更清晰
- 可同时展示检测结果、问答内容、知识命中与实时指标

## 五、技术路线概述

系统总体链路如下：

1. 摄像头与麦克风采集现场输入。
2. 板端执行视觉识别与离线语音识别。
3. 将用户问题与当前视觉上下文结合，送入本地大模型与 RAG 检索链路。
4. 输出专业化回答与处置建议。
5. 通过语音播报和图文界面完成结果呈现。

这是一个典型的“感知 - 推理 - 执行”边缘 AI 闭环应用。

## 六、正式演示入口

```bash
cd /mnt/ssd/spacemit_project
bash scripts/launch_voice_guided_demo.sh
```

## 七、演示交互方式

常用交互方式包括：

- 语音唤醒后直接提问
- 进入不同巡检场景进行现场问答
- 中文快捷语音指令

推荐测试语句：

1. `当前画面检测到了什么缺陷？`
2. `请解释一级检测和二级判定结果。`
3. `这种缺陷可能带来什么风险？`
4. `根据知识库，应该先做什么处理？`
5. `总结一下`
6. `保存截图`

## 十、仓库不包含内容

以下内容未纳入 Git 仓库，这是有意为之：

1. `.venv/` 等本地 Python 运行环境。
2. `/mnt/ssd/models` 下的正式模型文件。
3. 大体积运行时权重，如部分 `.onnx`、`.bin` 文件。
4. 可重建的索引缓存、日志、临时输出和运行缓存。
5. 板端临时调试文件、异常产物和本地备份文件。

因此，本仓库是“竞赛最终提交版源码工程”，不是整盘板端 SSD 镜像。
